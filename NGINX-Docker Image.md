# NGINX Docker Image Commands
1. Pull the Official NGINX Image

To pull the latest NGINX image from Docker Hub.

docker pull nginx

You can also pull a specific version (tag):

docker pull nginx:1.21
2. Run NGINX in a Container

To start a new container from the NGINX image and run it in detached mode.

docker run -d --name my-nginx-container -p 8080:80 nginx

Explanation:

-d: Run the container in detached mode (in the background).

--name my-nginx-container: Assign a name to the container.

-p 8080:80: Expose port 80 inside the container to port 8080 on your host.

Access the NGINX server via http://localhost:8080.

3. Verify the Running NGINX Container

To check if the NGINX container is running and listening on the desired port.

docker ps

You should see an output like:

CONTAINER ID   IMAGE     COMMAND                  CREATED         STATUS         PORTS                  NAMES
4f7bc3c4c5f2   nginx     "/docker-entrypoint.…"   2 minutes ago   Up 2 minutes   0.0.0.0:8080->80/tcp   my-nginx-container
4. Access the NGINX Container Logs

To view the logs generated by the NGINX container.

docker logs my-nginx-container

To follow the logs in real-time:

docker logs -f my-nginx-container
5. Run NGINX with Custom Configuration

To use a custom NGINX configuration file, you can mount your nginx.conf file as a volume:

docker run -d --name my-nginx-container -v /path/to/nginx.conf:/etc/nginx/nginx.conf -p 8080:80 nginx

This will replace the default NGINX configuration with your custom configuration located at /path/to/nginx.conf on your local machine.

6. Serve Custom Content with NGINX

To serve your custom HTML or static files, mount a directory from your local machine to the container:

docker run -d --name my-nginx-container -v /path/to/your/html:/usr/share/nginx/html -p 8080:80 nginx

This will serve your local HTML files at http://localhost:8080.

7. Inspect the NGINX Container

To get detailed information about the running NGINX container.

docker inspect my-nginx-container

This provides information about networking, volumes, environment variables, and more.

8. Connect to NGINX Container Shell

To access the container and interact with the file system, you can use:

docker exec -it my-nginx-container bash

Now you can make changes to the container directly.

9. Stop the NGINX Container

To stop the running NGINX container.

docker stop my-nginx-container
10. Remove the NGINX Container

To remove the stopped container from your system.

docker rm my-nginx-container
11. Remove the NGINX Image

If you no longer need the NGINX image, you can remove it from your system.

docker rmi nginx
12. Create a Docker Network for NGINX and Other Containers

If you want to connect multiple containers (e.g., NGINX and a backend service), you can create a Docker network.

docker network create my-network

Then, run the NGINX container connected to that network:

docker run -d --name my-nginx-container --network my-network -p 8080:80 nginx

This allows communication between my-nginx-container and other containers within my-network.

13. Configure NGINX for Reverse Proxy (Example)

To configure NGINX as a reverse proxy, you need to update the nginx.conf file and then mount it as a volume. Example of a simple reverse proxy configuration:

In your nginx.conf:

server {
    listen 80;
    server_name localhost;

    location / {
        proxy_pass http://backend-container:8081;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

Run the NGINX container with this config:

docker run -d --name my-nginx-reverse-proxy -v /path/to/nginx.conf:/etc/nginx/nginx.conf -p 8080:80 --network my-network nginx

Now, NGINX will forward requests to backend-container on port 8081.

14. Enable HTTPS on NGINX (With SSL)

To use SSL, you’ll need to mount SSL certificates into the container.

docker run -d --name my-nginx-ssl -v /path/to/certificates:/etc/ssl/certs -p 443:443 nginx

You’ll need to configure NGINX for SSL by updating the nginx.conf file to use the certificates.

15. Scaling NGINX with Docker Compose

If you want to scale NGINX containers with Docker Compose, create a docker-compose.yml file:

version: '3'
services:
  nginx:
    image: nginx
    ports:
      - "8080:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
    networks:
      - my-network
networks:
  my-network:
    driver: bridge

Run Docker Compose:

docker-compose up -d
